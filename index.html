What is this Repose thing anyway?

Repose is a Reverse Proxy built on J2EE Servlet Filters, providing:

  • Reusable features that save devs time in development
  • Faster product deliveries with easily templated configs
  • A layer of abstraction from underlying systems
  • A product that is Battle Tested
  • A system with Proven Performance
  • An open source, Racker supported product
  • Some use Repose to abstract out one or two common tasks instead of copy/pasting them.
  • Internally, DevOps have basic Chef, Puppet, and Ansible templates.
  • We have our own RPM and Deb repository that can be added for easy installation

These aren't just your parent's Filters.

  • The Filters have a little extra awareness and reliance on the Context.
  • The Context provides special services:
    • DataStore
    • HTTP Connection Pooling
    • Centralized Response Modification
  • DataStore
    • Local OR Distributed
    • Currently only uses Ehcache
    • Pure JSR-107 on backlog (swapping)
  • HTTP Connection Pooling
    • Individual settings
    • Supports special cases (e.g. long timeout)
    • Default is one pool
  • Centralized Response Modification
    • Provides consistent access to the Context for creating Response messages

Repose Safety Features

Denial of Service attacks attempt to overwhelm the origin service with a deluge of traffic and ultimately your server becomes unreachable to any traffic.

Man in the Middle attacks take advantage of encryption vulnerabilities for the sake of rendering your service inoperable.

  • Repose Rate Limiting protects the Origin Service from repeated requests that could overwhelm the OS.
  • Repose Caching protects dependent services, such as Keystone (Identity) through aggressive caching of previous responses.
  • Repose makes use of configurable SSL encryption to ensure traffic going through Repose is secure from man in the middle attacks.
  • Repose Translation filter prevents XML attacks.

Misc Interesting Facts…

  • Repose has been architected in a way that supports many platforms - runs in a jvm in front of apps in many languages.
  • Scalable horizontally - multi-nodel
  • Supports pluggable, custom filters… WSGI/Jython bridge in the works with Jim Baker
  • Repose is the authorization component for Identity. Identity provides the data, Repose enforces the policy.
  • You can switch out the Identity Provider (IdP) backend by switching Repose filters/configs.
  • Horizontal scaling allows for more complex configurations to maintain high throughput.
